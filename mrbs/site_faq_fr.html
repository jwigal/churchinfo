Comment est-on identifié ?

L'application peut utiliser plusieurs méthodes d'authentification dont LDAP, Netware, et SMB. Contactez votre administrateur système si vous avez des problèmes pour entrer dans l'application. Certaines fonctionnalités sont réservées à certains utilisateurs, d'autres utilisateurs auront le message Vous n'avez pas les droits suffisants pour faire une modification. Voyez votre administrateur système si cela ne fonctionne pas bien pour vous. Si l'application utilise l'authentification LDAP, vous devez vous connecter en utilisant le même identifiant et mot de passe que vous utilisez pour récupérer votre courrier électronique, par exemple "Mark Belanger" "MonMotDePasse". Pourquoi ne puis-je pas supprimer ou modifier une réservation ?

Pour pouvoir supprimer ou modifier une réservation, vous devez être authentifié comme étant la même personne qui a créé la réservation. Contactez soit un des administrateurs où la personne qui a créé la réservation pour la supprimer ou la modifier. Haut de page

Comment créer une réservation qui se répète ?

Après avoir cliqué sur l'heure désirée, l'écran de réservation s'affiche. Choisissez le Type de Périodicité approprié. La salle sera réservée à la même heure, jusqu'à la Date de Fin de Périodicité, et seulement pour les jours spécifiés par le Type de Périodicité.

Le type Jour crée une réservation chaque jour. Le type Semaine répète la réservation pour les jours de la semaine que vous avez coché sur la ligne Jour. Par exemple, pour une périodicité hebdomadaire programmée pour chaque lundi, jeudi et vendredi, cochez ces jours sur la ligne Jour. Si vous ne cochez pas de jours, la périodicité aura lieu chaque semaine le même jour que celui initialement choisi. Le type Mois crée une périodicité le même jour chaque mois, par exemple le 15 de chaque mois. Le type Année crée une périodicité le même jour du même mois, par exemple chaque 15 mars. Le type Mois, même jour semaine crée une périodicité un jour par mois, le même jour de la semaine et avec la même position ordinale de la semaine dans le mois. Utilisez ce type pour créer des périodicité du type le premier lundi, le second jeudi ou le quatrième vendredi de chaque mois par exemple. N'utilisez pas ce type pour des réservations après le 28ème jour du mois. Enfin le type tous les n Semaines est le même type que le type semaine mais permet de spécifier un intervalle de semaines, par exemple tous les mardi et jeudi toutes les 3 semaines.

Comment supprimer une seule réservation faisant partie d'une périodicité ?

Choisissez la salle/jour/heure que vous voulez supprimer et choisissez Effacer une réservation. Attention si vous modifiez ensuite la périodicité les réservations supprimées individuellement reviendront. Haut de page

Comment réserver simultanément des salles sur différents sites ?

Ce n'est pas possible. Pour le moment, l'application ne peut pas enregistrer simultanément une réservation pour 2 salles différentes. Vous devez créer les réservations séparément. Assurez vous que les salles sont disponibles aux même périodes avant de faire la réservation. Haut de page

Je ne peux pas créer la réservation, j'ai le message "trop de réservations"!

Il est impossible de créer plus de 365 réservations pour une réunion. Cette limitation est nécessaire pour des raisons de sécurité. Il est possible en cas de besoin d'augmenter cette limite. Haut de page

Que se passe t'il si plusieurs personnes enregistrent la même réunion ?

Réponse rapide : la première personne qui clique le bouton Enregistrer gagne.
En fait, techniquement l'application utilise une base de donnée relationnelle multi-threaded, Multi-Utilisateurs qui peut gérer plusieurs milliers d'utilisateurs simultanés. Haut de page

Quelle différence entre "Interne" et "Externe" ?

Par défaut, MRBS utilise deux types de réunions. "Interne" correspond à des réunions d'employés. "Externe" correspond à tout autre type, des clients, des vendeurs, des investisseurs, etc. Selon vos besoins, il est possible de créer jusqu'à 10 types de réunion. Sur la vue principale du calendrier, les réunions sont surlignées par une couleur correspondante à leur type, et une légende de tous les types définis figure en bas de la vue principale du calendrier. Comment cela fonctionne t'il et qui l'a développé ?

L'application Meeting Room Booking System est un logiciel libre de droits dont les sources sont disponibles, qui est distribué selon la Gnu Public License(GPL). Cela signifie que que vous pouvez modifier, distribuer et utiliser ce logiciel gratuitement.

Cette application est écrite principalement dans le langage PHP, qui est un langage de programmation libre qui peut être mixé dans les pages webs de façon similaire au concept des "active server pages" de Microsoft. PHP est particulièrement bon dans l'accès aux bases de données.

La base de donnée utilisée par l'application est soit MySQL, soit PostgreSQL. MySQL est un serveur de base base de données SQL (Structured Query Language) robuste, Multi-Utilisateurs, multi-threaded et très rapide qui est aussi sous licence GPL. PostgreSQL est un serveur de base de données relationnelle SQL libre à base d'objets, doté de nombreuses caractéristiques et multi-utilisateurs.

L'application peut fonctionner sur de nombreuses plate-formes, y compris sur des architectures PC utilisant le système d'exploitation Linux.  Linux est un système d'exploitation libre et gratuit, similaire à unix.

Le serveur web utilisé est encore un autre logiciel libre et gratuit. Le serveur web Apache est le serveur web le plus populaire au monde.

En conclusion :
Chaque composant de ce système, du système d'exploitation à l'application, est complètement gratuit et libre, y compris le code source.
